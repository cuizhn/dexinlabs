## 3. `app/modules/content/` — Content Engine（内容引擎核心模块）

> 架构分层：**Page → Composable/Query → API (server) → Service → Repository → Drizzle → Neon**。
> 引擎内部分为 **Data Side**（数据） 和 **Render Side**（渲染），通过 `boot.ts` 的两阶段启动分别独立加载。

### 3.1 入口 & 启动编排

| 文件 | 作用 |
|------|------|
| `boot.ts` | **统一启动编排入口**，提供三函数：`bootDataSide()` / `bootRenderSide()` / `bootContentEngine()`（两阶段串行）。仅做依赖编排，无实现。 |

### 3.2 `bootstraps/` — 各组件初始化（boot 调用的实现文件）

| 文件 | 侧别 | 作用 | 依赖约束 |
|------|------|------|----------|
| `database.ts` | Data | 注册 Source（neon-drizzle） + Query（lazyQuery） | 只导入 data 契约，不碰 Parser/Renderer |
| `parser.ts` | Render | 注册 Markdown Parser + Frontmatter + Math | 只导入 render 契约，不碰 DB |
| `transformers.ts` | Render | 注册 7 个 Transformer（按 order 排序） | 只导入 render 契约 |
| `renderer.ts` | Render | 注册 VueRenderer（VNode 输出） | 只导入 render 契约 |

### 3.3 `contracts/` — 契约层（接口 + 断言，零实现）

**Data Side 契约**：
| 文件 | 定义 |
|------|------|
| `data.ts` | 聚合导出：Source / Loader / Query 契约 + DATA_CONTRACTS 枚举 |
| `Source.ts` | 纯数据抓取三方法：`findOne / findAll / count` |
| `Loader.ts` | 业务聚合（跨 Source 多个调用组装实体），并定义领域类型：`Course / Chapter / Lesson / Exercise / Asset` |
| `Query.ts` | 对外查询门面：`getCourse / getChapter / getLesson / getExercise / listChapters` |

**Render Side 契约**：
| 文件 | 定义 |
|------|------|
| `render.ts` | 聚合导出：Parser / Transformer / Renderer 契约 + RENDER_CONTRACTS 枚举 |
| `Parser.ts` | 解析接口：`parse(raw)` → `{ frontmatter, ast }`，定义 AST Node 类型 |
| `Transformer.ts` | 转换接口：`transform(astNode, context)`，定义 TocEntry、HeadingInfo、LinkInfo 等 |
| `Renderer.ts` | 渲染接口：`render(ast, context)` → `VNode \| string`，定义 VNode、RenderedOutput |

### 3.4 `core/` — 引擎核心（IoC + Pipeline + Facade）

| 文件 | 作用 |
|------|------|
| `registry.ts` | **IoC 注册中心**：所有 Source/Loader/Parser/Transformer/Renderer/Query 都在此注册。提供 `register*()` + `get*()` 系列。是两侧共享的唯一组合根。 |
| `engine.ts` | **统一 Facade**：`getEngine()` 返回 `ContentEngineFacade`，对外暴露 `.data.*`（数据操作） + `.render.*`（渲染操作）两大命名空间，以及 shortcut（`getChapter / renderContent` 等）。 |
| `pipeline.ts` | **组合调度层**：根据入参自动分流到 Data Pipeline 或 Render Pipeline。提供 `runPipeline()` + `createPipeline()`。 |
| `dataPipeline.ts` | **Data Pipeline**：`runDataPipeline(operation, params)` → Loader → Source 操作。 |
| `renderPipeline.ts` | **Render Pipeline**：`runRenderPipeline(content, opts)` 固定流程：Parser → Transformer × N（按 order） → Renderer。提供 `renderToHTML() / renderToVNode()` 快捷方法。 |

### 3.5 `source/` — Data Side：数据源实现

| 文件 | 作用 |
|------|------|
| `index.js` | 工厂 `createSource(type, deps, opts)`，仅保留 database/neon/neon-drizzle/markdown/filesystem 类型，统一返回 DatabaseSource |
| `database/DatabaseSource.js` | SourceContract 实现：通过 Drizzle 调 Neon PostgreSQL，提供 `findOne / findAll / count` |

### 3.6 `loader/` — Data Side：加载层（业务聚合，TS）

| 文件 | 作用 |
|------|------|
| `course.ts` | `loadCourse(slug)`：组装 Course Tree（Course → Chapters → Lessons 嵌套） |
| `chapter.ts` | `loadChapter(slug)`：Chapter + 关联 Exercise 列表 + 关联 Lessons |
| `lesson.ts` | `loadLesson(slug)`：Lesson 详情 + 所属 Chapter 信息 |
| `asset.ts` | `loadAsset(slug)`：单个资源文件 |

### 3.7 `repositories/` — Data Side：Repository 层（Drizzle CRUD + 典型查询，TS）

严格按 **Repository + Service 分层**：

| 文件 | 对应表 | 核心方法 |
|------|--------|----------|
| `CourseRepository.ts` | `courses` | `findAll / findBySlug / upsertBySlug / ...` |
| `ChapterRepository.ts` | `chapters` | `findAll / findBySlug / findByCourseId / upsertBySlug / ...` |
| `LessonRepository.ts` | `lessons` | `findAll / findBySlug / findByChapterId / upsertBySlug / ...` |
| `ExerciseRepository.ts` | `exercises` | `findAll / findBySlug / findByChapterId / upsertBySlug / ...` |
| `AssetRepository.ts` | `assets` | `findAll / findBySlug / upsertBySlug / ...` |
| `index.ts` | 聚合 | `createRepositories()` 返回所有 Repository 实例 |

> 约束：Repository 只做纯 CRUD + 典型查询（带 where/orderBy/limit），**不做跨 Repository 聚合**，聚合放 Service 层。内部使用 `_getDb()` 方法懒加载 Drizzle db 实例，避免模块顶层初始化时环境变量未就绪。

## 4. `server/` — Nitro Server 层

> Nitro 自动扫描目录：`server/api/`（路由）、`server/utils/`（auto-import）、`server/middleware/`、`server/plugins/`。所有文件只运行在 SSR / Vercel Serverless 环境。

### 4.1 `server/api/` — API 路由（REST 风格）

五层调用链最后一环：**API → Service → Repository → Drizzle → Neon**。API 路由**只调 Service，不直接碰 DB**。

| 路由 | 对应 Service | 作用 | 错误处理 |
|------|-------------|------|----------|
| `course/index.get.js` | CourseService | GET `/api/course`：返回全部课程列表 | 404 无数据 |
| `chapter/index.get.js` | ChapterService | GET `/api/chapter`：返回全部章节（含筛选） | 404 无数据 |
| `chapter/[slug].get.js` | ChapterService | GET `/api/chapter/:slug`：章节详情 + Lessons + Exercises | 400 缺 slug / 404 不存在 |
| `lesson/[slug].get.js` | LessonService | GET `/api/lesson/:slug`：课时详情 + 所属 Chapter | 400 缺 slug / 404 不存在 |
| `exercise/[slug].get.js` | ExerciseService | GET `/api/exercise/:slug`：练习详情 + 所属 Chapter | 400 缺 slug / 404 不存在 |

> 约束：每个路由必须显式处理 **400（缺参数）** 和 **404（资源不存在）**，不能让错误冒泡成通用 500。

### 4.2 `server/utils/` — Server 层共享工具

| 文件 | 作用 |
|------|------|
| `db.ts` | 数据库工具：`useDb()` + re-export `drizzle/db.ts` 的 `db / getDb / createDb / closeDb / schema`，以及 `tables`（schema 表对象）。Server API 路由如果需要直接访问 db 从这里拿。 |

## 6. `content/` — Content Source of Truth（Markdown 真源）

> **Source of Truth 规则**：内容唯一真源是此目录的 Markdown/YAML 文件。数据库是缓存/查询层，`npm run sync` 将此目录 **幂等 upsert** 到 Neon。Runtime 严禁读 `content/` 目录。

```
content/
├── course/
│   └── course.yml                  # 课程元数据（title / summary / order / cover）
├── chapter/
│   ├── abc.yml                     # 章节元数据
│   └── quadratic-equation-in-one-unknown.yml
├── lesson/
│   ├── quadratic-equation-in-one-unknown-intro.md      # 课时正文（含 frontmatter）
│   ├── quadratic-equation-in-one-unknown-formula.md
│   ├── quadratic-equation-in-one-unknown-solve.md
│   ├── quadratic-equation-in-one-unknown-apply.md
│   └── quadratic-equation-in-one-unknown-exercise.md
└── sync.js                         # 同步脚本：fast-glob 扫描 → 解析 → 调 Repository upsert
```

### 同步脚本：`content/sync.js`

- 入口：`npm run sync`（package.json 定义）
- 流程：`扫描 course/chapter/lesson 三目录 → js-yaml 解析 YAML / Markdown frontmatter + body → 调对应 Repository upsertBySlug() → 打印同步统计`
- 幂等：按 slug 唯一索引 upsert，重复执行不产生脏数据

## 8. 关键调用链示意

### 8.1 页面渲染流程（SSR + Client）

```
  Page (pages/course/[chapter]/index.vue)
    │
    ├─ 调用 useChapter(slug)  ←─────────── 自动导入（query/useChapter.js）
    │        │
    │        └─ useAsyncData + $fetch(`/api/chapter/${slug}`)
    │                 │
    │                 ▼
    │           Nitro: server/api/chapter/[slug].get.js
    │                 │
    │                 ├─ getRouterParam(event, 'slug')   ← 400 缺参
    │                 └─ ChapterService.getChapterWithLessonsAndExercises(slug)
    │                      │
    │                      ├─ ChapterRepository.findBySlug()  ← 404 不存在
    │                      ├─ LessonRepository.findByChapterId()
    │                      └─ ExerciseRepository.findByChapterId()
    │                           │
    │                           ▼
    │                      drizzle/db.ts (Proxy 懒初始化)
    │                           │
    │                           ▼
    │                      Neon PostgreSQL
    │
    └─ <MarkdownRenderer :body="chapter.body" />
              │
              └─ contentEngine.renderContent(raw)
                   │
                   ├─ Parser: markdown 解析 → AST
                   ├─ Transformer × 7（order 升序）
                   └─ Renderer: VueRenderer → VNode → DOM
```

### 8.2 内容同步流程

```
  npm run sync
       │
       ▼
  content/sync.js
       │
       ├─ fast-glob 扫描 content/course, content/chapter, content/lesson
       ├─ js-yaml + frontmatter 解析 → { slug, title, body, ... }
       ├─ CourseRepository.upsertBySlug(item)
       ├─ ChapterRepository.upsertBySlug(item)
       └─ LessonRepository.upsertBySlug(item)
             │
             ▼
        drizzle/db.ts → Neon
```

## 9. 硬约束速查（避免反复踩坑）

1. **srcDir: 'app/'**：`~/` 别名 = `app/`。plugins、components、pages、layouts、assets、composables 必须放 `app/` 内。
2. **Content SoT**：`content/` 是唯一真源，Runtime 严禁直接读取。同步用 `npm run sync`，数据库字段不得覆盖 Markdown 独有的字段。
3. **五层调用链**：Page → Composable → API → Service → Repository → Drizzle。严禁跨层（API 不直连 Drizzle，Page 不调 Service）。
4. **API 错误处理**：每个路由必须显式 throw `createError({ statusCode: 400 })`（缺参数）和 `404`（资源不存在）。
5. **Drizzle db 懒加载**：Repository 用 `_getDb()` 方法或 Proxy db，**禁止模块顶层初始化 Pool**（否则 Vercel 环境变量未就绪就会崩）。
6. **TS 导入规范**：TS 文件间 import 保留 `.js` 扩展名（ESM 兼容）；TS 导入对方模块不写扩展名由 Nuxt/Nitro 解析。
7. **Vercel 部署**：`npm run build` 后用 `npx vercel deploy --prebuilt`，不要重新 build（避免 Vercel 环境变量时序问题）。
