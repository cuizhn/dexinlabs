# 得心实验室（Dexin Labs）

# Lesson Design Specification（LDS）

## Version 1.5（Learning-driven Edition）

> **版本**：V1.5
> **定位**：Learning-first（学习优先）
> **适用范围**：所有 Lesson、Learning Unit、练习、诊断、推荐系统。
> **核心原则**：所有系统都服务于学习者，而不是服务于课程。

> **每一节课，都应该帮助孩子真正理解知识，而不是仅仅完成学习任务。**
> **V1.5 与 V1 最大的区别：V1 关注如何组织课程；V1.5 开始关注学习者如何真正学会。**

---

# 一、设计理念（Design Philosophy）

## 1.1 我们不是在设计课程，而是在设计学习过程

课程只是知识的载体。

真正需要设计的是：

> 学习者如何一步一步建立理解。

因此：

任何页面、任何数据库、任何系统，都必须回答一个问题：

> 它是否帮助学习者理解？

如果不能，就不应该存在。

---

## 1.2 学习目标

我们的目标不是：

> 教会一个知识点。

而是：

> **帮助孩子建立自主学习能力。**

因此，每一节课都应该培养：

- 提出问题的能力
- 思考能力
- 表达能力
- 迁移能力
- 反思能力

知识只是载体。

---

## 1.3 学习遵循认知规律，而不是教材顺序

教材顺序：

```
定义
  ↓
性质
  ↓
例题
  ↓
练习
```

得心实验室：

```
发现问题
  ↓
产生需求
  ↓
提出猜想
  ↓
建立概念
  ↓
形成定义
  ↓
理解规律
  ↓
迁移应用
  ↓
反思总结
```

学习不是接受定义。

学习是建立认知。

---

## 1.4 学习体验

课程应该让学生产生：

> **"原来如此！"**

而不是：

> **"终于背下来了。"**

真正的理解，

应该伴随着：

- 好奇
- 成就感
- 自信

---

## 1.5 陪伴原则

课程不是老师讲课。

而是：

> 一个学习伙伴。

课程语言应该：

像一个耐心的大哥哥、

大姐姐。

而不是：

考试说明书。

---

## 1.6 所有页面都根据学习状态动态变化

页面不是固定内容。

页面应该反映学习者当前状态。

例如：

**第一次学习：**
```
介绍 Topic
  ↓
开始学习
```

**学习过程中：**
```
继续学习
  ↓
当前位置
  ↓
下一步建议
```

**已经掌握：**
```
知识网络
  ↓
复习建议
  ↓
练习巩固
```

页面属于学习者。

不是属于课程。

---

# 二、学习模型（Learning Model）

学习模型采用以下层级：

```
Domain
  ↓
Topic
  ↓
Lesson
  ↓
Learning Unit
  ↓
Concept
```

---

## Domain

学习领域。

**作用**：组织 Topic，帮助学习者建立整体认知。

**不是**：知识边界、教材章节。

**未来**：主要作为知识地图过滤条件存在。

---

## Topic

学习主题。

围绕一个核心问题组织。

例如：
```
函数
一次函数
二次函数
圆
概率
```

**Topic 是学习入口，也是学习状态的核心对象。**

---

## Lesson

一次完整学习活动。

**Lesson 不等于教材课时。**

**目标**：帮助学习者完成一次完整理解。

**构成**：可以包含一个或多个 Learning Unit。

---

## Learning Unit

一次认知推进。

**Learning Unit 是教学最小单位。**

**特点**：不是固定长度、不是固定数量。

**要求**：每一个 Unit 都应该完成一次认知提升。

例如：
```
提出问题
  ↓
观察现象
  ↓
提出猜想
  ↓
验证
  ↓
形成理解
```

如果 Concept 较难，可以拆分多个 Unit。

---

## Concept

Concept 是认知节点。

**特点**：理解的最小对象，也是未来诊断系统的最小粒度。

**关系**：
- 一个 Lesson 可以涉及多个 Concept
- 一个 Concept 也可能跨多个 Lesson
- Concept 之间存在依赖关系，未来逐步形成 Concept Graph

---

# 三、课程设计原则（Design Principles）

每一个 Lesson 都必须回答六个问题：

---

## ① 为什么学习？

不要直接给定义。

首先建立学习需求。

学习者必须知道：为什么需要这个知识。

---

## ② 当前真正的问题是什么？

不是老师的问题。

而是学习者的问题。

例如：
```
为什么要学习函数？
为什么需要方程？
```

课程必须围绕问题展开。

---

## ③ 真正困难在哪里？

真正困难不是知识点。

而是认知断点。

课程需要尽量找到：为什么学习者容易不会。

---

## ④ 如何循序渐进？

不要一次讲很多。

每一次只推进一个认知。

Learning Unit 必须连续，不能跳跃。

---

## ⑤ 如何判断真正理解？

理解不是会做题。

真正理解应该至少满足：

- 能够解释
- 能够举例
- 能够迁移
- 能够解决新的问题

---

## ⑥ 如果不会，是哪里不会？

Lesson 结束时，必须明确：哪些 Concept 没建立。

**未来用于诊断。**

---

# 四、课程结构（Lesson Structure）

推荐采用以下学习流程：

```
为什么学习
  ↓
今天解决什么问题
  ↓
生活中的例子
  ↓
自己猜一猜
  ↓
观察规律
  ↓
形成概念
  ↓
正式定义
  ↓
完整例题
  ↓
变化例题
  ↓
总结规律
  ↓
费曼讲解
  ↓
少量练习
```

根据内容可调整，但整体遵循：从经验到抽象。

---

## STEP 1 为什么要学习（Motivation）

回答：

> 今天为什么学习这个？

不是："因为考试要考。"

而是：它能够解决什么问题？

例如：

学习方程：

不是："今天学习一元一次方程。"

而是：

> 怎样求一个不知道的数量？

---

## STEP 2 从真实问题开始（Real World Problem）

每一节课必须从：

生活、游戏、故事、实验、图片、动画开始。

**不要**：定义。

例如：
```
妈妈买了3支铅笔和一本练习册，共花了18元。练习册多少钱？
```

孩子开始思考。

---

## STEP 3 自己猜（Prediction）

不要马上讲。

先问：
```
你觉得呢？
你会怎么做？
还能怎样想？
```

哪怕猜错，也是学习。

---

## STEP 4 建立规律（Discovery）

规律不是老师告诉。

而是：一步一步发现。

例如：
```
数字变化
  ↓
观察
  ↓
猜测
  ↓
验证
  ↓
得到规律
```

规律应该：被发现。

不是：被宣布。

---

## STEP 5 正式定义（Formal Concept）

定义永远放后面。

因为：定义只是规律的名字。

例如：
```
前面发现：不知道的数量可以用字母表示。
  ↓
最后：这种字母叫未知数。
```

孩子不会觉得陌生。

---

## STEP 6 为什么这样（Reason）

不仅告诉：怎么做。

还要告诉：为什么。

例如：

- 为什么移项要变号？
- 为什么负负得正？
- 为什么分数要通分？

**这一部分最重要。**

---

## STEP 7 一个完整例题（Worked Example）

要求：完整思考过程。

**不是**：只有答案。

包括：
```
为什么第一步这样？
为什么不用另一种方法？
哪里容易错？
```

---

## STEP 8 渐进练习（Progressive Practice）

练习不要重复。

建议：

| 题序 | 类型 | 说明 |
|------|------|------|
| 第一题 | 标准 | 基础练习 |
| 第二题 | 数字变化 | 巩固基础 |
| 第三题 | 场景变化 | 应用迁移 |
| 第四题 | 故意设计易错 | 识别陷阱 |
| 第五题 | 综合迁移 | 形成能力 |

这样才能形成能力。

---

## STEP 9 常见错误（Common Mistakes）

不是：最后告诉答案。

而是：提前告诉容易掉坑。

例如：
```
很多同学会这样写……
为什么？
哪里错？
```

帮助学生建立"自我检查"意识。

---

## STEP 10 一句话总结（Key Takeaway）

Lesson 最后：只能保留一句话。

例如：
```
未知数，只是暂时不知道的数量。
```

学生应该记住：思想。

不是：整页文字。

---

# 五、学习科学原则（Learning Science）

整个 LDS 推荐融合以下理论。

---

## 5.1 心流（Flow）

课程难度保持：
```
70% 已有能力 + 30% 新挑战
```

不能太简单，也不能太难。

---

## 5.2 费曼学习法（Feynman）

每节课最后必须增加：
```
如果你来教别人，你会怎样解释？
```

输出就是学习。

---

## 5.3 刻意练习（Deliberate Practice）

练习必须有目标，不是数量。

每一道题应该知道：训练什么。

---

## 5.4 检索练习（Retrieval Practice）

课程中定期关闭提示，让学生自己回忆。

例如：
```
刚才那个公式，还能写出来吗？
```

---

## 5.5 间隔重复（Spaced Repetition）

知识应该再次出现：
```
明天 → 三天 → 七天 → 一个月
```

自动复习。

---

## 5.6 元认知（Metacognition）

课程最后增加：
```
今天：
我真正学会了吗？
哪里还不会？
为什么不会？
```

帮助学生认识自己的学习。

---

# 六、诊断设计原则（Diagnosis）

诊断不是考试。

诊断是帮助学习。

因此，诊断必须：

- **及时**：在学习过程中随时诊断
- **准确**：定位到具体 Concept
- **温和**：鼓励式反馈，不打击信心
- **可解释**：告诉学习者为什么不会

---

诊断重点不是：
```
不会一次函数
```

而是：
```
不会变量
不会对应关系
不会坐标
不会函数思想
```

诊断应该逐步定位真正原因。

---

推荐必须接受诊断验证。

如果推荐效果不好，系统应自动修正推荐方向。

形成闭环：
```
推荐
  ↓
学习
  ↓
诊断
  ↓
修正推荐
```

---

# 七、学习进度设计（Progress）

Progress 不只是记录完成情况。

而是记录学习过程。

未来重点关注：

| 维度 | 内容 |
|------|------|
| 当前位置 | 当前 Topic、Lesson、Learning Unit |
| 学习难点 | 高频错误、停留时间 |
| 学习记录 | 最终用于帮助学习者、家长、推荐系统、诊断系统 |

---

# 八、内容规范（Content Rules）

每节课只解决一个核心知识点。

避免一节课讲四五件事情，认知负荷过高。

---

# 九、语言规范（Language）

始终使用鼓励式语言。

| 推荐 | 避免 |
|------|------|
| 很好。 | 错误。 |
| 已经接近答案了。 | 不对。 |
| 再想一步。 | 重新做。 |
| 你发现了吗？ | |

教育不是裁判。

---

# 十、AI 时代新增原则（LDS 2.0 预留）

未来 AI 加入后：

AI 不能直接给答案。

应该逐步引导：

```
第一步：提醒
第二步：提示
第三步：举例
最后：展示完整过程
```

让孩子始终保持思考。

---

# 十一、页面设计原则（Page Design）

---

## 11.1 首页设计原则

首页不是课程目录。

首页是学习入口。

进入首页：检查 Progress

**没有学习记录**：
```
开始学习 → 选择学习阶段 → 完成少量基础问题 → 建立初步学习画像 → 开始学习
```

**有学习记录**：
```
继续学习 → 直接返回当前学习位置
```

首页永远优先帮助学习。

---

## 11.2 知识地图设计

知识地图不是课程目录。

知识地图帮助学习者建立整体认知。

**默认**：直接展示全部 Topic。

**Domain**：不作为主体，仅作为过滤器。

**Topic 根据学习状态动态显示**：
- 已掌握
- 正在学习
- 待学习

知识地图帮助学习者理解：自己在哪里，下一步去哪里。

---

## 11.3 Topic 页面设计

Topic 页面根据学习状态动态变化。

**待学习**：
```
介绍 Topic → 说明学习意义 → 开始探索
```

**学习中**：
```
显示 Lesson → 显示当前位置 → 提示继续学习 → 适时推荐练习
```

**已掌握**：
```
显示知识概览 → 显示知识联系 → 建议复习 → 建议练习
```

Topic 不是目录，而是学习控制中心。

---

## 11.4 Lesson 页面设计

Lesson 页面强调沉浸学习。

**去除干扰**：
- 默认不显示顶部导航
- 仅保留返回 Topic

**页面布局**：

| 位置 | 内容 |
|------|------|
| 左侧 | 待解决的问题、当前目标 |
| 中间 | Markdown 正文（最大阅读区域） |
| 右侧 | 智能提示（依据 Progress 与 Diagnosis 动态生成） |
| 右下角 | 我的理解（学习笔记，鼓励用自己的语言表达） |

未来支持关联 Topic、Concept 与相关知识，而不是传统章节导航。

---

# 十二、产品设计原则（Product Design）

得心实验室不是内容平台，而是学习平台。

因此，所有模块都必须服务学习：

```
Content
  ↓
Progress
  ↓
Diagnosis
  ↓
Assessment
  ↓
Recommendation
```

所有系统共同形成：

```
观察
  ↓
判断
  ↓
调整
  ↓
继续学习
```

学习不是一次完成，而是不断修正、不断理解、螺旋上升。

---

# 十三、课程完成标准（Definition of Done）

一节课完成，不是因为内容写完，而是因为它满足以下标准：

| 检查项 | 要求 |
|--------|------|
| 学习动机 | 能回答"为什么学" |
| 问题情境 | 有真实或贴近生活的问题 |
| 主动思考 | 学生在获得答案前至少思考一次 |
| 概念形成 | 定义建立在规律发现之后 |
| 原理解释 | 解释"为什么"，而非只讲"怎么做" |
| 渐进练习 | 包含基础、变化、易错和迁移 |
| 主动输出 | 包含费曼式表达任务 |
| 主动回忆 | 包含检索练习 |
| 自我反思 | 引导学生评估自己的理解 |
| 语言风格 | 鼓励、陪伴、尊重孩子 |

---

# 十四、V1.5 当前阶段原则（最重要）

未来系统会包含：
- Progress Engine
- Diagnosis Engine
- Recommendation Engine
- Assessment Engine
- Concept Graph

但 **V1 阶段不追求全部实现**。

当前工作的重点只有两个：

1. **建立稳定、正确、可扩展的学习模型**
2. **编写高质量课程，在实际写作中不断验证"什么是真正的理解""什么是真正的循序渐进""什么才值得诊断"**

**架构不是最终目标。**

**课程不是最终目标。**

**真正的目标，是帮助学习者一步一步建立理解。**

---

# 十五、得心实验室课程设计信条（LDS Creed）

> **我们不追求让孩子学得更多，而追求让孩子真正学会。**
> **我们不急于告诉答案，而愿意陪伴孩子发现答案。**
> **我们相信，真正的教育不是灌输知识，而是点燃理解、培养思考、建立自信。**
> **当孩子能够独立面对未知、乐于探索、持续成长时，我们的课程才真正完成了它的使命。**

---

> **LDS V1.5 是一个长期演进的规范。**
> **它不断吸收教育心理学、认知科学、数学教育研究，以及在陪伴孩子学习过程中积累的真实经验。**
> **未来发展到 LDS 3.0、4.0……，LDS 本身就会成为得心实验室最宝贵、也最难复制的成果。**
